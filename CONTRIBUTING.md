# Contributing to Langtool CLI Installer

Thank you for your interest in contributing to the Langtool CLI Installer! This document outlines the process for reporting issues, proposing improvements, and submitting changes.

## Code of Conduct

We expect all contributors to foster a welcoming, inclusive, and respectful environment. Please treat fellow contributors with courtesy and kindness. Reports of unacceptable behavior can be sent to the maintainers at support@example.com.

## Getting Started

1. **Fork the repository** and clone your fork locally.
2. **Create a new branch** for your work using a descriptive name (e.g., `feature/add-new-flag`).
3. **Install dependencies** required for development:
   ```bash
   pip install -r install/requirements.txt
   ```
4. **Set up your environment** according to the instructions in the [`README.md`](README.md).

## Reporting Issues

- Use the GitHub Issues tracker to report bugs or request enhancements.
- Provide detailed information including steps to reproduce, expected behavior, and relevant logs or screenshots.
- Tag your issue with appropriate labels, if possible.

## Making Changes

1. **Plan your change** and discuss significant ideas with maintainers via an issue or discussion.
2. **Follow existing code style** and update documentation when required.
3. **Include tests** for new features or bug fixes when applicable.
4. **Run the test suite** before submitting your changes:
   ```bash
   pytest
   ```

## Commit Guidelines

- Write clear, concise commit messages that describe the change.
- Group related changes into a single commit when possible.
- Reference related issues in your commit messages (e.g., `Fixes #42`).

## Pull Request Process

1. Ensure your branch is up to date with the latest `main` branch.
2. Fill out the pull request template, summarizing your changes and their motivation.
3. Link to any related issues and provide screenshots or logs as needed.
4. Address automated review comments and requested changes promptly.
5. A maintainer will merge your pull request once it passes review and checks.

## Documentation

- Update `README.md` or other documentation files when you change functionality.
- Include examples or usage updates to help users understand new behavior.

## Questions

If you have any questions or need clarification, please open a discussion thread or contact the maintainers at support@example.com.

We appreciate your contributions and look forward to collaborating with you!
